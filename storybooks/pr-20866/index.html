Summary
fix(settings): match passkey password-fallback error handling to sign-in
Description
Because:
- The passkey password fallback diverged from the sign-in page: incorrect
  passwords showed in a blocking banner, Continue could be re-clicked with
  the same wrong password, and editing the field did not clear the error.

This commit:
- Routes password-validation and incorrect-password errors to an inline
  field tooltip and all other errors to the banner; keeps Continue disabled
  after a wrong password until the field is edited and clears the tooltip on
  edit; shows a client-side "Valid password required" tooltip on empty
  submit.
- Has onContinue return a classified error so the component owns the
  tooltip/banner surfaces instead of the container.
- Relabels the post-reauth navigation-failure metric reason to
  'navigation_error' (was 'server_error') and drops the unreachable
  PASSWORD_REQUIRED (1010) routing branch.
