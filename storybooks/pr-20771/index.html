Summary
fix(settings): migrate dev-server config to webpack-dev-server v5
Description
Because:
- webpack-dev-server v5 removed onBeforeSetupMiddleware/onAfterSetupMiddleware
  and the top-level `https` option. Under the v5 bump the fxa-settings dev
  server failed to start, so the Playwright functional tests failed because
  the app never served.

This commit:
- Fold the onBefore/onAfter middleware setup (evalSourceMap, user proxy setup,
  redirectServedPath, no-op service worker) into setupMiddlewares.
- Replace `https: getHttpsConfig()` with the v5 `server` option (http / https /
  {type:'https',options}). Verified webpack-dev-server 5.2.5 accepts the config
  via its constructor schema validation for both http and https paths.
