Summary
feat(passkeys): track PRF-less registration retries via Glean
Description
Because:
 - The silent retry that drops the WebAuthn PRF extension on Windows Hello
   (FXA-13991) emitted no telemetry, so we could not measure how often PRF
   must be dropped or know when the retry can safely be removed.

This commit:
 - Adds the account_pref.passkey_create_retry_without_prf_request event with
   reason (triggering DOMException name) and outcome (success/failure) extra
   keys, plus the event.outcome string metric.
 - Surfaces the retry from createCredentialWithPrfFallback via an onPrfFallback
   callback, keeping the fallback util free of any metrics dependency.
 - Emits the event from PagePasskeyAdd; covers the new paths in tests.
