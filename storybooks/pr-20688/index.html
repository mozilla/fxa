Summary
feat(dev): speed up local stack startup with a fast build path
Description
Because:
 - `yarn start mza` was slow: the start tasks pulled the full production
   build (including tsc) on every boot, though dev servers compile from
   source at runtime.

This commit:
 - Adds a `build-fast` Nx target that skips `build-ts` for app services
   while still building dist-loaded libs, and routes `yarn start mza` and
   `yarn restart mza` through it via FXA_BUILD_TARGET=build-fast.
 - Drops `build` from the `start`/`restart` target dependencies; the build
   now runs explicitly in pm2-all.sh (full build by default), so `yarn
   start` still does a full build and CI, which calls `build` directly, is
   unaffected.
 - Hardens local startup: Docker daemon preflight, labeled URL checks with
   curl timeouts, a non-fatal redis readiness gate, and a db-patcher
   expected-count derived from the migrations layout.
 - Parallelizes the fxa-shared ESM/CJS tsc build to cut critical-path time.
 - Fixes the l10n-gettext-extract workflow to read .nvmrc from the
   checked-out fxa-code/ path.
